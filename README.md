# claude-setup

> A Claude Code plugin that automatically configures the Claude environment for new projects

One command — `/claude-setup:run <project description>` — and you get:
- Interactive requirements gathering and `docs/requirements.md` generation
- Automatic research for relevant official/community skills
- Auto-configuration of `CLAUDE.md`, agents, skills, memory, and permissions
- Stack-aware agents dynamically generated for your project (e.g. backend developer, frontend developer)
- Implementation roadmap created in Plan Mode, saved to `docs/plan.md`, and indexed in long-term memory

---

## Installation

### Remote (no clone required)

```bash
# 1. Register the marketplace
/plugin marketplace add jayl2kor/claude-setup

# 2. Install the plugin
/plugin install claude-setup@jayl2kor-claude-setup
```

### Local

```bash
# From a local clone
git clone https://github.com/jayl2kor/claude-setup.git
claude plugin install ./claude-setup
```

## Usage

```bash
# Run in a new project directory
/claude-setup:run "FastAPI-based user authentication microservice"
/claude-setup:run "React + TypeScript e-commerce platform"
/claude-setup:run "Go CLI tool for Kubernetes cluster management"

# Start without a description (Claude will ask)
/claude-setup:run
```

### Phases

| Phase | What happens | Output |
|-------|-------------|--------|
| 1 | Interactive requirements gathering (3 question clusters) | Summary table |
| 2 | Generate requirements document | `docs/requirements.md` |
| 3 | Research bundled/external skills and recommend | Recommendation table |
| 4 | Generate Claude environment files | CLAUDE.md, agents, skills, memory, settings.json |
| 5 | Create implementation plan in Plan Mode | `docs/plan.md` (registered in long-term memory) |

User confirmation is required before each phase transition.

## What's Bundled

### Skills (8)

| Skill | Purpose |
|-------|---------|
| `skill-creator` | Official skill/agent authoring methodology |
| `deep-research` | Systematic technical research methodology |
| `backend-patterns` | REST API, clean architecture, CQRS, DB patterns |
| `frontend-patterns` | Component design, state management, performance |
| `code-formatting` | Lint/formatting per language (Python, JS/TS, Go, Rust) |
| `antipattern-detection` | GoF antipatterns, SOLID violations, language-specific pitfalls |
| `e2e-testing` | Playwright best practices, test pyramid, CI integration |
| `security-review` | OWASP Top 10, auth/authz, secrets management |

### Agents (5 bundled + dynamic)

| Agent | Purpose |
|-------|---------|
| `code-reviewer` | Code quality, DRY, maintainability |
| `security-reviewer` | OWASP vulnerability detection, security code review |
| `ui-designer` | Visual design — uses `ui-ux-pro-max-skill` when installed, falls back to MD3/HIG/WCAG built-in knowledge |
| `ux-designer` | UX design — uses `ui-ux-pro-max-skill` when installed, falls back to Nielsen/WCAG/cognitive load built-in knowledge |
| `domain-researcher` | Search and evaluate external official/community skills |

**Dynamic agents** are generated by Phase 4 based on your project stack:

| Stack / Domain | Auto-generated Agent |
|----------------|---------------------|
| API server (FastAPI, Express, Spring, etc.) | `backend-developer` — domain-researched REST/architecture expert |
| React / Vue / Next.js / frontend framework | `frontend-developer` — component design and state management expert |
| ML / AI pipeline | `ml-engineer` — data pipeline, model evaluation, deployment patterns |
| Security-critical (finance, healthcare, payments) | `security-engineer` — threat modeling, compliance, secure design |
| Special domain (legal, logistics, etc.) | Custom domain expert agent |

Each dynamic agent is written after deep-researching the domain's architecture patterns, best practices, and common pitfalls — not just a generic role description.

> Bundled agents/skills are applied automatically during setup and can also be used independently.

## Generated Files

```
your-project/
├── CLAUDE.md                        # Project guide + quality gates
├── docs/
│   ├── requirements.md              # Gathered requirements
│   └── plan.md                      # Implementation roadmap (Phase 5)
└── .claude/
    ├── agents/
    │   ├── code-reviewer.md
    │   ├── backend-developer.md     # Stack-specific (if applicable)
    │   ├── frontend-developer.md    # Stack-specific (if applicable)
    │   └── <custom-agent>.md
    ├── skills/
    │   └── <custom-skill>/
    └── settings.json                # Permissions + hooks

~/.claude/projects/<project>/memory/
├── MEMORY.md                        # Index with pointer to docs/plan.md
└── <seed files>.md
```
